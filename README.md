## 🎯 Objectif
Travail d'évolution/amélioration d'UX/UI d'interfaces médicales SaaS à destination de patients et de professionnels de santé. Création d'une application mobile sur base du Saas, pour les patients.

## 📦 Périmètre (MVP)
- Interface mobile patient pour alertes et suivi
- Dashboard praticien pour visualiser les données patients
- Traitement de flux de données capteurs
- Génération d’alertes simples et compréhensibles
- Protection des données et chiffrement

## Descriptif des fichiers


### Vision

| Fichier                | Objectif                  | Contenu attendu                                    | Bonnes pratiques              |
| ---------------------- | ------------------------- | -------------------------------------------------- | ----------------------------- |
| `vision.md`            | Définir la vision produit | Problème, solution, proposition de valeur, cible   | Court, clair, orienté produit |
| `personas.md`          | Décrire les utilisateurs  | 2-3 personas avec besoins, frustrations, objectifs | Réalistes, pas caricaturaux   |
| `problem-statement.md` | Formaliser le problème    | Contexte, problème précis, impact                  | Éviter les solutions ici      |

### 📋 Fonctionnel
| Fichier                   | Objectif                     | Contenu attendu                       | Bonnes pratiques         |
| ------------------------- | ---------------------------- | ------------------------------------- | ------------------------ |
| `user-stories.md`         | Décrire les besoins          | User stories + critères d’acceptation | Format INVEST            |
| `use-cases.md`            | Détailler les interactions   | Scénarios d’usage détaillés           | Inclure acteurs + flux   |
| `parcours-utilisateur.md` | Visualiser l’expérience      | Étapes du parcours utilisateur        | Peut inclure schémas     |
| `regles-metier.md`        | Formaliser la logique métier | Règles, contraintes, conditions       | Précis et sans ambiguïté |


### 🎨 UX-UI

| Fichier              | Contenu attendu                                 |
| -------------------- | ----------------------------------------------- |
| `wireframes.md`      | écrans simplifiés (croquis, schémas)            |
| `maquettes.md`       | lien Figma + captures + explications            |
| `design-system.md`   | couleurs, typo, composants                      |
| `parcours-ux.md`     | expérience utilisateur détaillée (optionnel)    |

### 🏗 Technique

| Fichier               | Objectif               | Contenu attendu                       | Bonnes pratiques      |
| --------------------- | ---------------------- | ------------------------------------- | --------------------- |
| `architecture.md`     | Vue globale du système | Description architecture + schéma     | Rester compréhensible |
| `choix-techniques.md` | Justifier les technos  | Stack + arguments (pourquoi ce choix) | Comparer options      |
| `data-model.md`       | Modéliser les données  | Entités, relations, schéma            | Clair + cohérent      |
| `api-spec.md`         | Définir les API        | Endpoints, inputs/outputs, exemples   | Style REST clair      |
| `contraintes.md`         | Définir les contraintes        | Hardware, software, numéro de version  | Style REST clair      |
| `cout de run.md`         | Définir les couts de fonctionnement        | Estimation sur une année de fonctionnement  | Style REST clair      |


### 🧱 Architecture (C4)
Basé sur le modèle C4 model

| Fichier         | Objectif       | Contenu attendu        | Bonnes pratiques        |
| --------------- | -------------- | ---------------------- | ----------------------- |
| `contexte.md`   | Vue externe    | Acteurs + système      | Très simple             |
| `conteneurs.md` | Structuration  | Frontend, backend, DB… | Responsabilités claires |
| `composants.md` | Détail interne | Modules principaux     | Pas trop granulaire     |
