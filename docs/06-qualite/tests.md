# Tests

## Stratégie
- Tests fonctionnels : vérifier les parcours patient et praticien.
- Tests d’intégration : valider l’acheminement des données capteurs vers le dashboard.
- Tests UX : s’assurer que l’interface mobile reste simple et compréhensible.
- Tests de sécurité : vérifier chiffrement et protection des données.

## Cas de test
- Connexion / inscription patient et praticien.
- Réception d’une alerte de surpression.
- Affichage du score de santé.
- Modification de paramètres par le praticien.
- Vérification de la confidentialité des données.

## Priorités
1. Fonctionnalité de suivi patient
2. Alerte de surpression
3. Dashboard praticien
4. Protection des données
